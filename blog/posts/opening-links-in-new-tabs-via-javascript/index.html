Opening Links in New Tabs via Javascript

In Markdown, the easiest way to add a link to a line of text is like this:

In this example, the word GitHub will be a clickable link to https://github.com. And in most Markdown interpreters, including on GitHub, that link will open in the same tab that you‚Äôre currently in. And if you‚Äôre like me, this is irritating.

So, I endeavored to find the best way to set my Markdown links to open in new tabs. By default, in Markdown with kramdown (my selected Markdown-to-HTML converter), there‚Äôs only two ways to make this happen:

After several pull requests of me changing my mind about how to write links, and doing massive search-replaces in my codebase, I found a different kind of solution‚Ä¶ a solution that involves Javascript. Static sites (like mine) can use Javascript to do view-based logic and methods right in front of the user. It‚Äôs unwise to incorporate a lot of Javascript on a single page because it‚Äôll slow down loading and operation times, but it‚Äôs how I did simply functionality, like showing the comments on the bottom of each page or loading the Feather icons. And so of course, somebody has written this blog post which nicely provides a Javascript solution for opening every external URL in a new tab. This script is what is suggested to be placed at the bottom of our HTML files:

<script type="text/javascript">
  function externalLinks() {
    for(var c = document.getElementsByTagName('a'), a = 0; a < c.length; a++) {
      var b = c[a];
      b.getAttribute('href') && b.hostname !== location.hostname && (b.target = '_blank')
    }
  };
  externalLinks();
</script>

It‚Äôs very simple, but yet, it covers almost exactly what I need. On every page that‚Äôs loaded, it will go through my converted and compiled HTML, find each attribute that contains a link, and will guarantee that each EXTERNAL link will open to a new tab, and that each INTERNAL link opens to the current tab. Voil√†!

But there‚Äôs one piece missing. That‚Äôs for exceptions. Every now and then‚Äîsuch as writing this blog post‚ÄîI want an EXTERNAL link that will open in the current tab. So, I need a way to exclude those few links from opening in a new tab. The answer comes from changing the for loop to look like this:

var links = document.getElementsByTagName('a');

for(counter = 0; counter < c.length; counter++) {
  var link = links[counter];

  if (link.target == '_self') {
    link.getAttribute('href') && link.hostname !== location.hostname && (link.target = '_self');
  } else {
    link.getAttribute('href') && link.hostname !== location.hostname && (link.target = '_blank');
  };
};

So, if the link in Markdown is written explicitly requesting the target="_self", then honor that. So the following link would open in the current tab:

This will be a link to <a href="https://github.com/" target="_self">GitHub</a>.

Similarly, if a link is explicitly written with target="_blank", no matter whether external or internal, the link should always open that in a new tab. If nothing is set for the target at all, then open in a new tab if the link is external.
